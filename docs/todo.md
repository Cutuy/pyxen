## Next big bet: shell-based thin wrapper

- [ ] Ship **one universal escape hatch per primitive** — `shell` impl that wraps any CLI
  - Per-method command config (`put`, `get`, `query`, `delete` → CLI invocation)
  - I/O modes: `stdin` / `parse` (raw, json, jsonl, stdout-json) / `result_path`
  - Error model: non-zero exit → primitive error with stderr captured
  - Async: `asyncio.to_thread` wrapper
  - Accepted trade-offs: ~10-50ms spawn cost, no streaming, fragile error handling
- [ ] `shell` impl for `storage`
- [ ] `shell` impl for `secrets`
- [ ] `shell` impl for `identity`
- [ ] `shell` impl for `pkg`

## Companion `http` impl

- [ ] Ship `http` impl for REST-first backends (Vault, Datadog, Honeycomb, AWS Secrets Manager, GCP IAM)
  - Same shape as `shell`: method, url, headers (`{env:VAR}` interpolation), parse, result_path
  - Together `shell` + `http` cover ~95% of production backends

- [ ] `http` impl for `storage` / `secrets` / `identity`
